# 老师任务与程序功能对应说明

## 任务（1）：调用 PYPOWER 进行潮流计算

实现位置：

- `pfagent/caseio.py`：读取 `.py`、`.m`、`.mat`、`.json` 算例。
- `pfagent/solvers.py`：通过 `pypower.api.runpf()` 执行潮流计算。
- `pfagent/cli.py`：提供 `inspect`、`run`、`doctor` 命令。

演示命令：

```powershell
.\.venv\Scripts\python.exe -m pfagent run .\examples\case9_demo.py --engine pypower
```

## 任务（2）：LLM 检查数据和分析工程不合理情况

实现位置：

- `pfagent/validators.py`：确定性检查数据和工程约束。
- `pfagent/llm.py`：连接 DeepSeek、Kimi。
- `pfagent/agent.py`：把原始数据摘要、规则问题、求解尝试和数值诊断发送给 LLM。

检查内容：

- REF/PV/PQ 节点配置；
- 孤岛与缺失平衡节点；
- 母线电压上下限；
- 机组 P/Q、VG 限额；
- 支路端点、阻抗和容量；
- 发电能力与负荷；
- 线路过载；
- 发电机 Q 越限；
- PV/REF 达到 Q 限额后的 PV→PQ。

演示命令：

```powershell
.\.venv\Scripts\python.exe -m pfagent inspect .\examples\case3_bad_data.py
```

## 任务（3）：不收敛诊断、方案生成、修改并重新计算

实现位置：

- `pfagent/solvers.py`：初始牛顿雅可比矩阵秩、条件数和最小奇异值诊断。
- `pfagent/repairs.py`：数据修复、平坦启动、机组再调度、调压和负荷调整。
- `pfagent/agent.py`：LLM 候选动作和确定性后备流程。

后备流程：

```text
牛顿法筛查
→ Q 限额与 PV→PQ
→ 平坦启动
→ 快速解耦 XB
→ 快速解耦 BX
→ Gauss-Seidel
→ 机组有功再调度
→ 从同一原始基准搜索负荷比例
→ 重新检查电压、Q限额和线路容量
```

演示命令：

```powershell
.\.venv\Scripts\python.exe -m pfagent run .\examples\case3_q_limit.py --engine pypower --max-rounds 20
```

## 可行性的严格定义

程序不会把“牛顿法收敛”等同于“工程可行”。最终成功必须同时满足：

- 求解器数值收敛；
- 没有母线电压越限；
- 没有支路容量越限；
- 没有未处理的 QMAX/QMIN 越限；
- PV→PQ 状态已经过重新计算验证。

只有满足这些条件才生成 `final_feasible_case.py`。

## LLM 的职责边界

LLM 用于：

- 模仿工程师解释数据和结果；
- 判断更可能是数据错误、数值问题还是物理不可行；
- 对允许的修复动作排序；
- 生成结构化诊断意见。

LLM 不直接宣布方案有效。任何方案都必须重新调用 PYPOWER，并通过确定性工程约束检查。
